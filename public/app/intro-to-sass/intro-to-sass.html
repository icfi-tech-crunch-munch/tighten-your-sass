What is Sass

Sass = Syntactically Awesome StyleSheets

CSS Processors are CSS with special sauce that compiles down to plain old CSS that should work in any browser.

Less. vs. Sass vs. Stylus

Less

Less is "entry level" CSS enhancement. Lacks some of the control logic that Sass and Stylus have.

Less syntax follows CSS very closely. Brackets and semicolons are requred.

Bootstrap uses Less for version 3 and under, but is switching to Sass for v4. There is a popular mixin library called LESShat.

@base-border-color: #ccc;
@base-border-width: 1px;
@base-border-style: solid;

.panel {
    border: @base-border-width @base-border-style @base-border-color;
}

Compiles to:

.panel {
    border: 1px solid red;
}

Less.js is the core preprocesor. You could include the JavaScript library and include .less files into your project, but it's not recommended. There are wrappers that use Less.js to preprocess .less files into CSS.

Stylus

Stylus is a more "feature complete" CSS enhancement. It has control logic (if statements, loops) that Less lacks, as well as a few other benefits.

Stylus syntax is a lot more flexible than either Less or Sass. You can write in typical CSS style with brackets and semicolon or eschew those for whitespace significance, or some combination of both.

base-border-color = #ccc
base-border-width = 1px
base-border-style = solid

.panel
    border base-border-width base-border-style base-border-color

Compiles to:

.panel {
    border: 1px solid red;
}

Stylus is typically compiled via Node.js along with a task runner like grunt.

So far as I know there aren't any popular frameworks that use stylus. There is a popular mixin library called Nib.

Sass

Sass agressivly tries to match features with the competition. It has control logic (if statements, loops) that Less lacks, as well as a few other benefits.

Sass syntax is more flexible than either Less, but not as much as Stylus. You can write in typical CSS style with brackets and semicolon, but you have to name your file with .scss extention or you can eschew those for whitespace significance in a file with .sass extension. You cannot mix code styles.

Scss is typically more common, and all examples will follow that syntax style.

$base-border-color: #ccc;
$base-border-width: 1px;
$base-border-style: solid;

.panel {
    border: $base-border-width $base-border-style $base-border-color;
}

Compiles to:

.panel {
    border: 1px solid red;
}

Sass previously had a ruby on rails dependancy, but there is a port called libsass that is developed concurrently.

Frameworks that use scss include bootstrap 4 and Zurb Foundation. There are two popular mixin libraries: compass and bourbon.
